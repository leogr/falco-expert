### Contribution Types

The Falco community welcomes both code and non-code contributions:
- Code contributions to Falco and related projects
- Documentation improvements
- Blog posts
- Newsletter contributions
- Event organization and support
- Rules contributions

### Contributing Rules

To share custom Falco rules with the community:
1. Open a PR against the rules repository: https://github.com/falcosecurity/rules
2. Ensure rules follow the style guide
3. Include required testing and specify maturity level
4. Reviewers will assist throughout the process

**Source:** [share-rules.md](../../../refs/falcosecurity/falco-website/content/en/docs/contribute/share-rules.md)

### Building from Source

For developers wanting to build Falco:
1. Review the contributing guide
2. Follow build instructions in the developer guide
3. Build dependencies vary by Linux distribution (CentOS, Debian/Ubuntu, Arch Linux, Alpine, openSUSE supported)

**Source:** [source.md](../../../refs/falcosecurity/falco-website/content/en/docs/developer-guide/source.md)

### Governance Principles

- **Community Principles**: Openness, respect, transparency, diversity, and vibrancy
- **Leadership**: Maintainers of core repositories provide oversight and direction
- **Voting Power**: Limits set on per-company voting power
- **Technical Advisory Groups**: Can be formed for specific topics requiring expertise
- **Maintainer Lifecycle**: Clear path from contributor to maintainer to emeritus status
- **Decision Making**: Lazy consensus by default with defined voting rules
